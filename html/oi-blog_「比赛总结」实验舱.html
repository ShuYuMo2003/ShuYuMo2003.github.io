异或

给出一个大小为 nnn 的集合 SSS 和一个整数 xxx ，求 SSS 中有多少子集满足其中任意两个元素之间的异或和不小于 xxx 。对质数取模。 n≤106n \le 10^6n≤106

设 xxx 的二进制最高位为 kkk 。

考虑两个数字异或的情况，如果两个数字大于 kkk 的二进制位不一样，其异或结果一定不小于 xxx 。

那么如果我们把大于 kkk 的二进制位相同的数取出来，分成一个组，然后单独求出每组内能取出的子集个数，最后答案只需要对每个组能取出的子集个数自由组合即可。

考虑求出每组内能取出的子集个数。因为要求每组取出的子集中任意两个数的异或和所以这些数字在第 kkk 位的取值一定要两两不同，所以每一组内取出的合法子集大小只有 0,1,20, 1, 20,1,2 三种，前两种是平凡的，最后一种用 trie 维护即可。

计数

给定整数 xxx ，定义长度为 NNN 的序列 {ai}\{a_i\}{ai​} 的价值 f(a)f(a)f(a) 如下：

max⁡{L(a)−x,0} \max\{L(a)-x,0\} max{L(a)−x,0}

其中 L(a)L(a)L(a) 表示 aaa 最长的相同子段的长度。

给定整数 N,K,xN,K, xN,K,x ，求出长度为 NNN，各个元素在 [1,K][1,K][1,K] 中的所有整数序列的价值之和。大质数取模。 N≤106,K≤108,x≥0N \le 10^6, K \le 10^8, x \ge 0N≤106,K≤108,x≥0

首先一个显然的策略是枚举每个 L(a)L(a)L(a) 的可能取值 xxx，算方案数即可。

每次算方案数时可以设 dp[n][0/1]dp[n][0/1]dp[n][0/1] 考虑了长度为 nnn 的序列，达到了 / 未达到 xxx 的方案数。

这样就已经没办法优化了，可以考虑把等于的限制改宽改为大于的限制，注意到： max⁡{L(a)−x,0}=∑i=x+1n[L[a]≥i] \max\{L(a) - x, 0\}=\sum_{i=x+1}^{n}\left[L[a] \ge i\right] max{L(a)−x,0}=i=x+1∑n​[L[a]≥i] 类似地贡献转化手法还有这道题。可能是一种常见套路

问题转化为对于一个 iii 求有多少序列的连续子段长度大于 iii。

可以考虑补集转化，求有多少序列的连续子段长度小于 iii 的序列，只需要在每个位置枚举从这里出发连续字段有多长就可以了。（相当于一个存在性限制，转化为任意性限制，任意性限制显然结构更加简单）

可以设 dp[n]dp[n]dp[n] 表示长度为 nnn 的序列，最长相同子段的长度小于 xxx 的方案数，转移显然： dp[n]={knn<x(k−1)∑i=1x−1dp[n−i]otherwise. dp[n] = \begin{cases} k^n & n < x\\ (k-1)\sum_{i=1}^{x-1}\limits{dp[n-i]} & otherwise. \end{cases} dp[n]=⎩⎨⎧​kn(k−1)i=1∑x−1​dp[n−i]​n<xotherwise.​ 可以用前缀和优化。这样做复杂度仍旧是 O(n2)\mathcal{O}(n^2)O(n2) 的。

题解中给出了这样一份代码，其 dp[n]dp[n]dp[n] 的意义是长度为 nnn 的合法序列，且最后两个元素不同的方案数：

int getans(int n, int k, int x) {
    if (x == 1) return power(k, n);
    static int dp[MAXN]; dp[1] = k;
    for (int i = 2; i <= n; i++)
        if (i < x) dp[i] = 1ll * k * dp[i - 1] % P;
        else dp[i] = (1ll * k * dp[i - 1] - 1ll * (k - 1) * dp[i - x] % P + P) % P;
    return (0ll + power(k, n) - dp[n] + P + dp[n - x + 1]) % P;
}

按照题解的转移方式，其方程可以写作：（转移可以理解为把最后一个数字扩充，然后随便放上一个不一样的） dp[n]={kn=1dp[n−1]×ki<xdp[n−1]×k+(1−k)×dp[i−x]otherwise. dp[n]= \begin{cases} k &n=1\\ dp[n-1]\times k &i < x\\ dp[n-1]\times k + (1-k) \times dp[i-x] &otherwise. \end{cases} dp[n]=⎩⎨⎧​kdp[n−1]×kdp[n−1]×k+(1−k)×dp[i−x]​n=1i<xotherwise.​ 考虑到这里的方程转移比较单一，可以试图画出转移图，然后考虑贡献的传递，第一个点值为 kkk ，每一条转移边都是乘某一个值，且这个值的取值仅有 222 种。

如果一条从 111 到 nnn 的转移路径所经过的两种转移路径数量相同，那么他们的贡献也相同。枚举第二条转移路径的数量即可计算贡献。这样做的复杂度是 O(nx)\mathcal{O}(\frac{n}{x})O(xn​) 。根据调和级数的理论，这样的做法是 O(nlog⁡n)\mathcal{O}(n\log n)O(nlogn) 的。

C. 最大价值

A 君有 nnn 个物品，每个物品有两个属性 ai,bia_i,b_iai​,bi​，现在 A 君想从所有物品中挑选 kkk 个，并按一定顺序摆放好。

对于一个方案中，被摆在第 jjj 个位置（位置从 111 开始标号）的物品为 iii，它对这个方案产生的价值贡献为 ai⋅(j−1)+bia_i\cdot(j-1)+b_iai​⋅(j−1)+bi​，一个方案的价值和为它所含的所有物品的贡献之和。

现在 A 君想知道对于所有可能的 kkk（1≤k≤n​1\le k\le n​1≤k≤n​），在最优的选取以及摆放情况下能得到的方案价值和最大是多少。

接下来 nnn 行，每行两个整数 ai,bia_i, b_iai​,bi​。

100%100\%100% 的数据：n≤300 000n \le 300\,000n≤300000, 0≤ai≤1060 \le a_i \le 10^60≤ai​≤106, 0≤bi≤10120 \le b_i \le 10^{12}0≤bi​≤1012

输出 nnn 行，每行两个整数 ai,bia_i, b_iai​,bi​ ，表示 k=1,2,3,4,⋯ ,nk=1, 2, 3, 4, \cdots , nk=1,2,3,4,⋯,n 时的最大总价值。

对于一个大小为 kkk 的方案，如果确定了选择哪 kkk 个，就可以直接按照 aaa 排序，然后依次算出贡献（排序不等式）

那么显然可以考虑先对原物品按照的 aaa 排序，然后依次做背包即可。 dp[i][j]=max⁡{dp[i−1][j],dp[i−1][j−1]+ai×j+bi} dp[i][j] = \max\{dp[i-1][j], dp[i-1][j-1] + a_i\times j+b_i\} dp[i][j]=max{dp[i−1][j],dp[i−1][j−1]+ai​×j+bi​} 观察到一条结论，选定 kkk 个物品的答案，一定是在原来 k−1k-1k−1 个物品的答案基础上添加一个物品，而不会拿走。

如果考虑了 iii 种物品，那么对于选取 jjj 个物品的方案，一定是从选 j−1j-1j−1 个物品的方案添加一个物品得到的。也就是说，每新考虑一个物品，上面的转移一定是在某个 jjj 之前都是继承上一轮的答案，之后都是新增这个物品的答案，维护差分数组即可。可以平衡数维护每一个 jjj 处的答案的差分。

  \end{cases}
  \]
  考虑到这里的方程转移比较单一，可以试图画出转移图，然后考虑贡献的传递，第一个点值为
  \(k\) ，每一条转移边都是乘某一个值，且这个值的取值仅有 \(2\) 种。
  
  如果一条从 \(1\) 到 \(n\)
  的转移路径所经过的两种转移路径数量相同，那么他们的贡献也相同。枚举第二条转移路径的数量即可计算贡献。这样做的复杂度是
  \(\mathcal{O}(\frac{n}{x})\) 。根据调和级数的理论，这样的做法是
  \(\mathcal{O}(n\log n)\) 的。
  
  有一道平衡树优化 dp 转移的题目，比较有意思。

  \subsection{\texorpdfstring{\href{/pdf/20210310/C.pdf}{C.
  最大价值}}{C. 最大价值}}\label{c.-ux6700ux5927ux4ef7ux503c}
  
  A 君有 \(n\) 个物品，每个物品有两个属性 \(a_i,b_i\)，现在 A
  君想从所有物品中挑选 \(k\) 个，并按一定顺序摆放好。
  
  对于一个方案中，被摆在第 \(j\) 个位置（位置从 \(1\) 开始标号）的物品为
  \(i\)，它对这个方案产生的价值贡献为
  \(a_i\cdot(j-1)+b_i\)，一个方案的价值和为它所含的所有物品的贡献之和。
  
  现在 A 君想知道对于所有可能的
  \(k\)（\(1\le k\le n​\)），在最优的选取以及摆放情况下能得到的方案价值和最大是多少。
  
  接下来 \(n\) 行，每行两个整数 \(a_i, b_i\)。
  
  \(100\%\) 的数据：\(n \le 300\,000\), \(0 \le a_i \le 10^6\),
  \(0 \le b_i \le 10^{12}\)
  
  输出 \(n\) 行，每行两个整数 \(a_i, b_i\) ，表示
  \(k=1, 2, 3, 4, \cdots , n\) 时的最大总价值。
  
  对于一个大小为 \(k\) 的方案，如果确定了选择哪 \(k\) 个，就可以直接按照
  \(a\) 排序，然后依次算出贡献（排序不等式）
  
  那么显然可以考虑先对原物品按照的 \(a\) 排序，然后依次做背包即可。 \[
  dp[i][j] = \max\{dp[i-1][j], dp[i-1][j-1] + a_i\times j+b_i\}
  \] 观察到一条结论，选定 \(k\) 个物品的答案，一定是在原来 \(k-1\)
  个物品的答案基础上添加一个物品，而不会拿走。
  
  如果考虑了 \(i\) 种物品，那么对于选取 \(j\) 个物品的方案，一定是从选
  \(j-1\)
  个物品的方案添加一个物品得到的。也就是说，每新考虑一个物品，上面的转移一定是在某个
  \(j\)
  之前都是继承上一轮的答案，之后都是新增这个物品的答案，维护差分数组即可。可以平衡数维护每一个
  \(j\) 处的答案的差分。
  
  官方题解附有结论证明。
